Hermes Agent 非常创新，它可以**主动创建 Skill（技能）**，会在对话过程中，自动把值得复用的任务总结成一个小笔记（Skill），存进内部数据库，相当于自己写了一本专属秘籍。

它采用 **渐进式 + 模块化调用**，需要的时候只智能加载当前需要的部分，而不是一次性全塞进来。

这样做的好处是：
- 节省 token
- 记忆不会膨胀
- 注意力更集中
- 还能越用越聪明

本文就介绍下在**小龙虾养殖中心（LC-03）**使用 Hermes，得益于懒猫微服小龙虾养殖中心的强大基座，部署超方便，使用超简单~

> 推荐使用 LC-03 48G大内存，运行更快、更流畅、不卡顿，购机优惠咨询请加专业团队

## 前置准备

首先安装最新版**小龙虾养殖中心**，如果已经在用了，可以一键导入 OpenClaw 配置到 Hermes 中。

需要确保科学上网环境正常。

## 一键安装

进入小龙虾养殖中心，双击桌面上的终端按钮，复制下方命令，在终端中 `Ctrl + Shift + V` 粘贴，回车执行：

```bash
bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
```

安装程序会开始运行，如果失败可以检查科学上网环境。

进行到激活虚拟环境这一步时，可能会卡很久，这是正常的，请耐心等待。Permission denied 的提示可以忽略。

## 一键迁移 & 初始化

这里看你的需求，决定是否从小龙虾养殖中心的 OpenClaw 中导入配置，输入 `Y` 回车即可。

跟着向导执行快速初始化，然后设置模型接入点。部分供应商可能会导入失败，重新输入添加，后续重新配置下。

## 配置消息渠道

Hermes 暂无 WebUI，这里需要配置消息渠道：

如果你的 Telegram Bot 已经自动从 OpenClaw 配置中导入了，直接回车就行。

参考向导选择进行配置，稍后手动配对消息渠道 id。

## 启动

安装进入末尾，直接顺手启动。

## 配对消息渠道

随便发送消息给 Bot，点击配对命令，粘贴到终端进行配对。

## 守护进程（自启动）

在桌面上找到"额外扩展功能"，进入"自启动管理"，点击添加，填入命令：

```bash
hermes gateway run&
```

保存后重启小龙虾养殖中心，即可实现自启动。

重新配置好模型后，就可以正常和 agent 对话了！
